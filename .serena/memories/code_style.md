# TradingAgents 代码风格规范

## Python代码风格
- **遵循PEP 8标准**
- **类型注解**: 强制使用类型提示（Python 3.10+）
- **文档字符串**: 使用Google风格的docstring
- **导入顺序**: 标准库 -> 第三方库 -> 本地模块

## 命名规范
- **类名**: PascalCase (如 `TradingAgentsGraph`)
- **函数名**: snake_case (如 `create_market_analyst`)
- **常量**: UPPER_SNAKE_CASE (如 `DEFAULT_CONFIG`)
- **私有方法**: 前缀下划线 (如 `_update_current_report`)

## 项目结构规范
- **模块化设计**: 按功能划分目录结构
- **智能体分离**: 每个智能体独立文件
- **工具集中管理**: 统一的工具接口
- **配置外部化**: 使用配置文件管理参数

## 错误处理
- **异常捕获**: 使用try-except处理可预见错误
- **日志记录**: 使用logging模块记录关键信息
- **优雅降级**: 网络错误时使用缓存数据

## 性能考虑
- **异步处理**: 数据获取使用异步模式
- **缓存机制**: 重复数据请求使用缓存
- **内存管理**: 大数据处理时注意内存释放
- **API限制**: 遵守第三方API调用频率限制
